MindCorp ships with two functional page groups, **Principal 1** and **Principal 2**. These are template sections that demonstrate how to build and integrate new business modules into the portal using the standard Razor Pages pattern, permission checks, and neumorphic UI components.

## Principal 1

The Principal 1 group is gated by permissions with the prefixes `principal11.*` and `principal12.*`. The dropdown appears in the navbar when the user holds at least one matching claim.

## Adding a new functional module

<Steps>
  <Step title="Register the module">
    Go to **Seguridad â†’ MĂłdulos** and create a new module entry. Give it a unique key prefix (for example, `inventario`).
  </Step>
  <Step title="Create the Razor Page">
    Add a new `.cshtml` / `.cshtml.cs` pair under `Pages/`. In the page model's `OnGet`, check for `principal-level.consultar` using the `TienePermiso` helper and redirect to `/Index` if absent â€” matching the pattern in `PrincipalD1.cshtml.cs`.
  </Step>
  <Step title="Add permission checks to the view">
    In the `.cshtml` file, declare the `esAdmin` flag and `TienePermiso` helper at the top of a code block, then gate each action button behind the appropriate key (`.agregar`, `.editar`, `.eliminar`, `.detalle`).
  </Step>
  <Step title="Wire up the navbar">
    In `_Layout.cshtml`, add a new `var tieneMiModulo` check using `TieneAlgunPermiso("mimodulo.")` and render the corresponding `<li>` dropdown entry.
  </Step>
  <Step title="Assign permissions to profiles">
    In **Seguridad â†’ Permisos Perfil**, assign the new module's permission keys to the appropriate user profiles. The navbar entry will appear automatically for any user whose JWT claims include a matching `permiso` value.
  </Step>
</Steps>

<Tip>
  Every principal page follows the same three-part pattern: (1) a `consultar` guard in `OnGet` that redirects unauthorized users, (2) a static or API-backed data table in the view, and (3) per-row action buttons gated by individual permission keys. Copy this pattern when building real business modules.
</Tip>
