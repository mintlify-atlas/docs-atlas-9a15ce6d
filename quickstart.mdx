<Note>
  The MindCorp frontend has no database of its own. It requires the **WebCoorporativaAPI** backend to be running and reachable before you start the frontend.
</Note>

## Prerequisites

<CardGroup cols={2}>
  <Card title=".NET 10 SDK" icon="code">
    MindCorp targets `net10.0`. Install the .NET 10 SDK from [dotnet.microsoft.com](https://dotnet.microsoft.com/download).
  </Card>
  <Card title="WebCoorporativaAPI" icon="server">
    The companion backend API must be running. Note its base URL — you will need it during configuration.
  </Card>
</CardGroup>

## Setup

<Steps>
  <Step title="Clone the repository">
    ```bash
    git clone https://github.com/JesusTrz/WebCoorporativaFront.git
    cd WebCoorporativaFront
    ```
  </Step>

  <Step title="Configure appsettings.Development.json">
    Open `WebCoorporativa/appsettings.Development.json` and set the API base URL and your Cloudflare Turnstile keys.

    ```json appsettings.Development.json
    {
      "ApiSettings": {
        "BaseUrl": "https://<your-api-host>/api"
      },
      "Turnstile": {
        "SiteKey": "<your-turnstile-site-key>",
        "SecretKey": "<your-turnstile-secret-key>"
      },
      "DetailedErrors": true,
      "Logging": {
        "LogLevel": {
          "Default": "Information",
          "Microsoft.AspNetCore": "Warning"
        }
      }
    }
    ```

    | Key | Description |
    |-----|-------------|
    | `ApiSettings:BaseUrl` | Full base URL of the WebCoorporativaAPI, including the `/api` path segment. |
    | `Turnstile:SiteKey` | Public site key from your Cloudflare Turnstile widget. Used to render the CAPTCHA on the login page. |
    | `Turnstile:SecretKey` | Secret key used server-side to verify the Turnstile token. Keep this out of source control. |

    <Warning>
      Never commit real `SecretKey` values to version control. Use environment variables or a secrets manager in production.
    </Warning>
  </Step>

  <Step title="Run the application">
    From the `WebCoorporativa/` directory, start the dev server:

    ```bash
    dotnet run
    ```

    The application starts on `https://localhost:5001` (or the port shown in your terminal). Unauthenticated requests to `/` are automatically redirected to `/Login`.
  </Step>

  <Step title="Log in for the first time">
    Navigate to `/Login` in your browser.

    1. Enter your username and password.
    2. Complete the **Turnstile CAPTCHA** widget that appears on the form.
    3. Submit the form.

    The frontend calls `POST api/auth` on the backend, receives a JWT, and stores it in the server-side session. ASP.NET cookie authentication then tracks your session via the `WebCoorporativa.AuthCookie` cookie.

    <Tip>
      If the login page returns an error, confirm that `ApiSettings:BaseUrl` is correct and that the WebCoorporativaAPI is running and reachable.
    </Tip>
  </Step>

  <Step title="Explore the dashboard">
    After a successful login you are redirected to `/Index`. The top navigation bar renders menu items dynamically based on the permission claims decoded from your JWT:

    - **Principal 1** — visible when you hold `principal11.*` or `principal12.*` permissions.
    - **Principal 2** — visible when you hold `principal21.*` or `principal22.*` permissions.
    - **Seguridad** — visible when you hold `perfil.*`, `permisosperfil.*`, `usuario.*`, or `modulo.*` permissions, or when `esAdmin` is `true`.
    - **Mi Perfil / Cerrar Sesión** — always visible in the user dropdown.

    Admin users (`esAdmin: true`) see all menu sections regardless of individual permission assignments.
  </Step>
</Steps>
